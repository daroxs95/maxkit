  # Accessibility

  At ui toolkit level, by default critical molecules handles accessibility,
  but ultimately this depends a lot on context and content, and for that reason all components support standard aria
  attributes.

  ## Inclusive Design Guide

  The following tips are a starting point for creating accessible and inclusive experiences:
  - Alternative text for media content
  - Avoid images of text, since this text that cannot be translated, selected, or understood by assistive technology
  - Provide alternatives and/or descriptions for complex visualizations
  - Make sure text and background colors contrast sufficiently (Read more about why this matters)

  For engineers:
  - Make sure all content belongs to elements like (`header`, `footer`, `nav`, `main`, `article`)
